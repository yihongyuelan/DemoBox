Simplify and unify
To use the Facade Pattern, we create a class that simplifies and unifies a set of more complex classes that belong to some subsystem. Unlike a lot of patterns, Facade is fairly straightforward; there are no mind bending abstractions to get your head around. But that doesn't make it any less powerful; the Facade Pattern allows us to avoid tight coupling betwen clients and subsystems.

The Facade Pattern provides a unified interface to a set of interfaces in a subsystem. Facade defines a higher-level interface taht makes the subsystem easier to use.
